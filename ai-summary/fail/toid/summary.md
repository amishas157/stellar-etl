## Meta-Patterns

1. **Dead-Code Bugs Are Not Live Risks** (003, 004): Genuine logic bugs in utility functions (`IncOperationOrder`, `LedgerRangeInclusive`) that have zero production callers cannot corrupt exported data. Always grep for callers before escalating severity.

2. **Documented Capacity Limits Are Design Constraints** (002, 005, 018, 022): When a limit is explicitly documented with a timeline (170-year synthetic-offer ceiling, int32 TOID boundary) or protected by XDR caps (`MAX_OPS_PER_TX = 100`), treat it as an accepted design constraint rather than a bug. The test is whether wrong data can be produced today.

3. **Upstream Scope Classification Must Be Honored** (011, 012/013): Never synthetically derive an operation TOID for a row just because the surrounding transaction has one operation. Verify whether the upstream event producer explicitly marks the row as transaction-scoped (e.g., fee events, diagnostic events) — a null `operation_id` may be the correct signal.

4. **ETL–Horizon Convention Independence** (019, 020a, 020b, 021): The ETL defines its own BigQuery-oriented data contracts. Divergences from Horizon's API conventions (effect ID format, effect indexing, Soroban superset effects) are not automatically bugs. Consistent field naming, test coverage, schema comments, and golden files all signal intentional design.

5. **Codebase-Wide Parquet Null→Zero Pattern** (008, 009): Every `null.Int` / `null.String` in the JSON schema flattens to the zero value in the parquet schema throughout the codebase. No parquet scalar field uses OPTIONAL repetition. Individual per-field instances of this pattern are not independent bugs; the pattern itself is a deliberate architectural trade-off.

6. **TOID `+1` and Index Convention Audits** (001, 006, 010): When callers or exporters disagree on `+1` adjustments or index bases, trace the actual upstream ingest contract (`LedgerTransaction.Index` is 1-based) and all production call sites before concluding there is a mismatch. A different convention that produces correct output for all real callers is a design choice.

7. **Parquet Path Existence Precondition** (007): Before investigating nullable-TOID corruption on a parquet export path, confirm that the command actually calls `WriteParquet(...)`. A shared `--write-parquet` flag is not sufficient if the command body never exercises it.
