## Meta-Patterns

1. **Protocol 23 backward-compatibility aliasing**: When XDR renames a field (e.g. `BucketList*` → `SorobanState*`, `TotalByteSizeOfBucketList` → `TotalByteSizeOfLiveSorobanState`), both old and new column names are intentionally populated from the renamed field to preserve downstream BigQuery compatibility. This applies to config-setting exports, ledger exports, and potentially other entity types. Always check git history across protocol boundary commits before concluding that two columns sharing a source value is a bug.

2. **Dormant / unreachable code paths**: Several helpers (`GetAllHistory()`, `UpdateOrderbook()`, `transactionOperationWrapper.Details()`) contain real bugs (inverted loop condition, Go slice value-semantics, spelling errors) but are not wired into any active CLI export command. These should be tracked for re-opening when a caller is added, but they do not constitute live data-corruption hypotheses.

3. **Upstream-mirroring design**: The ETL intentionally mirrors the behavior of `stellar/go` ingest processors and Horizon's history DB conventions, including legacy quirks (`transaction_count` = successful only, `authorized_to_maintain_liabilites` without `_flag`). Before filing a divergence as a local bug, verify against the upstream processor and Horizon API definition.

4. **No XDR source, no viable finding**: A schema column that has no corresponding field in current generated XDR (e.g. `auto_bump_ledgers`) or whose only realistic source is the same field as another column (e.g. `soroban_resources_read_bytes`) is dead/legacy schema, not an active export bug, unless a distinct on-chain replacement value can be established.

5. **Uniform Parquet null-to-zero pattern**: The entire codebase converts `guregu/null` types to primitive zero values in Parquet output. This is an explicit architectural decision (zero `OPTIONAL` Parquet tags codebase-wide). Entity-specific "null collapse" hypotheses will always be NOT_VIABLE under this design.

6. **Cross-subsystem duplicate checking**: The same code path can be filed under different subsystem labels (`cli-commands`, `export-pipeline`, `utilities`). Before filing a hypothesis, check fail and success directories across all subsystems. Restored-key parquet gap and claimable-balance parquet skip were each filed three or more times across subsystem boundaries.

7. **Intentional omissions are design decisions**: Code comments that explicitly document "field X is not preserved" or "this is not needed in the current scope" are design decisions, not bugs — even when the omitted data would be useful to downstream consumers.
