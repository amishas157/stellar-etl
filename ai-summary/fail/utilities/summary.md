## Meta-Patterns

1. **SDK validates downstream**: An overwritten or discarded intermediate error is not a real data-integrity bug when the next SDK layer validates the same value and fails loudly before any export begins. (001, 011)

2. **Loud failure ≠ data corruption**: Any code path that returns an error or crashes before producing output does not satisfy the "incorrect but plausible output" criterion. Hardcoded paths, schema mismatches, and manifest conflicts that trigger startup errors are operational concerns, not silent-corruption findings. (009+010, 011, 016)

3. **Unreachable arithmetic/type triggers**: Overflow and underflow bugs require domain-range analysis confirming that production Stellar data can actually approach the boundary. Theoretical overflow (year 292B timestamps, sub-64 ledger underflow in dead code) does not qualify. (003, 004)

4. **Working-as-designed behaviors**: Several utility behaviors that look suspicious (ledger-2 default, testnet-wins on conflicting flags, muxed-to-canonical normalization, UPDATED for unchanged signers) are explicitly documented design decisions backed by consistent test expectations. (002, 005, 006, 018)

5. **Dead-code / no-live-caller rule**: Synthetic LCM builders and helpers without active export-path callers cannot produce wrong live output. Verify reachability before treating local logic as a bug. (003, 008)

6. **Protocol semantics constrain trigger space**: Classic ledger entry types (`DATA`, signers), pre-64 ledger ranges for orderbook, and non-overlapping sponsorship shapes are excluded by protocol/consensus rules, so code gaps affecting them have no realistic live trigger. (003, 014, 015)

7. **Cross-subsystem and cross-finding duplicate checking**: Bugs that span subsystems (captive-core routing, unchanged-signer UPDATED) may already be documented as success or fail findings elsewhere. Check all relevant subsystem files before filing a new hypothesis. (018, 019, 020)

8. **Legacy-alias columns**: Several output columns retain pre-Protocol-23 names as compatibility aliases. A suspicious duplicate-assignment in transform code is not a copy-paste bug if the current XDR no longer exposes a distinct source field. (012)
